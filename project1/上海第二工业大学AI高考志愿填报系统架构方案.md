# 上海第二工业大学AI高考志愿填报系统架构方案

## 1. 系统概述

### 1.1 系统目标
上海第二工业大学AI高考志愿填报系统旨在为考生提供智能、个性化的高考志愿填报指导服务，通过AI技术解答考生关于学校、专业、招生政策等方面的问题，帮助考生做出更加明智的志愿填报决策。

### 1.2 系统定位
- **服务对象**：高考考生及家长
- **服务内容**：学校信息查询、专业介绍、招生政策解读、志愿填报咨询
- **技术特点**：基于AI的智能问答系统，结合学校实际数据，提供个性化、精准的信息服务

### 1.3 设计原则
- **高可用性**：系统采用集群部署，确保服务稳定运行
- **易维护性**：模块化设计，便于系统升级和维护
- **安全性**：严格的安全策略，保护用户数据和系统安全
- **性能优化**：针对高并发场景进行优化，确保系统响应速度
- **符合高校信息化规范**：遵循学校现有信息化标准和规范

## 2. 技术架构

### 2.1 整体架构
系统采用三层架构设计：
1. **前端层**：Vue3 + TypeScript + Pinia + Axios + Element Plus
2. **后端层**：SpringBoot 3.2 + JDK17 + MyBatis-Plus + Redis
3. **AI服务层**：Python 3.10 + LangChain 0.2 + FastAPI + ChromaDB

### 2.2 技术栈明细
| 模块 | 技术选型 | 选型理由 |
| --- | --- | --- |
| 前端 | Vue3 + TypeScript + Pinia + Axios + Element Plus | 响应式强、组件化清晰、高校管理后台友好 |
| 后端 | SpringBoot 3.2 + JDK17 + MyBatis-Plus + Redis | 成熟稳定、与学校现有系统易集成、支持高并发 |
| AI服务 | Python 3.10 + LangChain 0.2 + FastAPI + ChromaDB | RAG流程标准化、向量检索高效、轻量易部署 |
| 嵌入模型 | BAAI/bge-small-zh-v1.5（本地HuggingFace） | 避免依赖DeepSeek嵌入API，降低成本+提升响应速度 |
| LLM调用 | DeepSeek-V2/V3（通过LangChain封装） | 自备密钥，仅用于生成答案，非嵌入计算 |
| 部署 | Docker + Docker Compose（开发）/ K8s（生产） | 环境隔离、便于教务处IT团队运维 |

## 3. 核心功能模块

### 3.1 前端系统

#### 3.1.1 页面结构
- **首页**：系统介绍、功能入口、热门问题推荐
- **专业库**：学校专业列表、专业详情、就业情况
- **问答界面**：智能问答交互、历史问题记录、答案分享
- **个人中心**：用户信息管理、收藏问题管理

#### 3.1.2 核心功能
- **智能问答**：用户输入问题，系统通过AI生成答案
- **专业查询**：按专业名称、类别等条件查询专业信息
- **志愿填报指导**：基于考生分数和兴趣推荐适合的专业
- **富文本答案渲染**：支持文本、图片等多种形式的答案展示
- **用户行为埋点**：收集用户操作数据，用于系统优化

### 3.2 后端系统

#### 3.2.1 网关层
- **认证**：用户身份验证、权限控制
- **限流**：防止恶意请求，保护系统安全
- **日志**：记录系统运行状态和用户操作

#### 3.2.2 业务层
- **会话管理**：维护用户会话状态
- **问题路由**：将用户问题路由到相应的处理模块
- **结果封装**：将AI服务返回的结果封装为前端可处理的格式

#### 3.2.3 服务层
- **AI服务调用**：与Python AI服务进行通信
- **本地缓存**：使用Redis缓存热点数据，提高响应速度
- **数据管理**：管理系统配置、用户数据等

### 3.3 AI服务系统

#### 3.3.1 LangChain RAG引擎
- **文档加载**：加载学校相关文档，如招生简章、专业介绍等
- **文本分割**：将文档分割为适合向量存储的小块
- **向量生成**：使用本地嵌入模型生成文本向量
- **检索增强**：根据用户问题检索相关文档，增强回答质量

#### 3.3.2 Chroma向量库
- **持久化存储**：将向量数据持久化到磁盘，确保数据安全
- **高效检索**：基于向量相似度进行快速检索

#### 3.3.3 DeepSeek API调用
- **环境变量管理**：通过环境变量管理API密钥，确保安全
- **答案生成**：调用DeepSeek API生成高质量答案

#### 3.3.4 多模态处理
- **文本答案**：生成结构化的文本答案
- **图片URL提取**：从答案中提取图片URL，用于前端展示

## 4. 数据流和核心业务流程

### 4.1 核心业务流程

1. **用户输入问题**：用户在前端界面输入关于学校、专业、招生政策等问题
2. **前端处理**：前端将问题发送到后端API
3. **后端处理**：
   - 网关层进行认证、限流和日志记录
   - 业务层进行会话管理和问题路由
   - 服务层调用AI服务
4. **AI服务处理**：
   - LangChain RAG引擎检索相关文档
   - 调用DeepSeek API生成答案
   - 多模态处理：提取图片URL
5. **结果返回**：
   - AI服务将结果返回给后端
   - 后端封装结果并返回给前端
   - 前端渲染富文本答案

### 4.2 数据流向

```
用户浏览器 → Vue3前端 → SpringBoot后端 → Python AI服务 → Chroma向量库 → DeepSeek API
                                 ↓
                             Redis缓存
                                 ↓
用户浏览器 ← Vue3前端 ← SpringBoot后端 ← Python AI服务
```

## 5. 部署方案

### 5.1 开发环境
- **Docker Compose**：管理多容器应用
- **服务组成**：
  - 前端容器（Nginx）
  - 后端容器（SpringBoot）
  - AI服务容器（Python）
  - Redis容器
  - Chroma向量库（持久化存储）

### 5.2 生产环境
- **Kubernetes**：容器编排和管理
- **服务组成**：
  - 前端服务（Nginx，多副本）
  - 后端服务（SpringBoot，多副本，负载均衡）
  - AI服务（Python，多副本）
  - Redis集群
  - Chroma向量库（持久化存储，高可用）

### 5.3 部署流程
1. **环境准备**：配置服务器环境，安装Docker/Kubernetes
2. **代码构建**：编译前端和后端代码，构建Docker镜像
3. **服务部署**：使用Docker Compose或Kubernetes部署服务
4. **配置管理**：通过环境变量管理配置信息
5. **监控设置**：配置系统监控和日志管理

## 6. 安全方案

### 6.1 安全架构
- **网络安全**：使用HTTPS协议，防止数据传输被窃取
- **身份认证**：基于JWT的身份验证机制
- **权限控制**：基于角色的访问控制（RBAC）
- **数据保护**：敏感数据加密存储

### 6.2 安全措施
- **API安全**：
  - 接口限流，防止恶意请求
  - 输入验证，防止SQL注入等攻击
  - 跨域资源共享（CORS）配置
- **数据安全**：
  - 用户数据加密存储
  - 敏感信息脱敏处理
  - 定期数据备份
- **系统安全**：
  - 定期安全扫描
  - 漏洞修复
  - 安全日志审计

## 7. 性能优化

### 7.1 前端优化
- **代码分割**：按需加载组件，减少初始加载时间
- **缓存策略**：缓存静态资源，减少网络请求
- **图片优化**：使用适当的图片格式和大小
- **渲染优化**：减少DOM操作，使用虚拟列表

### 7.2 后端优化
- **缓存机制**：使用Redis缓存热点数据
- **数据库优化**：合理设计数据库索引，优化查询语句
- **并发处理**：使用线程池处理并发请求
- **负载均衡**：分布式部署，负载均衡

### 7.3 AI服务优化
- **嵌入模型本地化**：使用本地嵌入模型，减少API调用
- **向量库优化**：合理配置Chroma向量库参数，提高检索速度
- **批处理**：批量处理相似请求，减少API调用次数
- **缓存策略**：缓存常见问题的答案，提高响应速度
